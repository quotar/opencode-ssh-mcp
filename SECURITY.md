# Security Policy

## Reporting a Vulnerability

If you discover a security vulnerability in opencode-ssh-mcp, please report it responsibly using one of the following methods:

- **Email**: [security@yourdomain.com] (replace with actual email)
- **GitHub**: Use the "Security" tab in the repository and create a "Private vulnerability disclosure"

Please include the following information in your report:
- Detailed description of the vulnerability
- Steps to reproduce
- Potential impact
- Possible solutions (if any)

**DO NOT report security vulnerabilities in public GitHub Issues.**

## Response Timeline

We commit to:
- Acknowledging receipt of vulnerability reports within 48 hours
- Providing initial response within 1 week, indicating how we plan to address the issue
- Providing regular updates on investigation and remediation progress

Reporters' privacy will be respected and protected.

## Responsible Disclosure

We follow responsible disclosure principles. Security researchers who report vulnerabilities responsibly will be publicly acknowledged (unless the researcher prefers anonymity) after the issue is resolved.

## Best Practices

### For Users
- Keep opencode-ssh-mcp updated to the latest version
- Regularly update dependencies
- Follow secure SSH configuration practices
- Use strong SSH keys

### For Developers
- Input validation and sanitization
- Secure coding practices
- Regular dependency auditing
- Penetration testing

## Supported Versions

| Version | Supported | Security Updates |
|---------|-----------|------------------|
| 1.x.x   | ✅         | ✓               |
| < 1.0   | ❌         | ✗               |

## Encryption Reporting

For highly sensitive information, you may use our PGP key for encrypted communication. The PGP key fingerprint will be published on our official website.

Thank you for helping keep opencode-ssh-mcp and its users safe!
